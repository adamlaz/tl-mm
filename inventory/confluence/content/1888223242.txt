Use this as a living reference for onboarding, reviews, and continuous improvement. Project Overview The MLR (Multi Location Report) Automation focuses on high-value regression coverage across critical user journeys, API contracts, and compliance rules to reduce cycle time and prevent release regressions. This document outlines the current automated test coverage for NEO Reporting. The tests validate core functionality across reports, including UI rendering, data visibility, filtering, navigation, and export behavior. Tests are executed in both: Single Location mode All Locations mode Test Scope and Coverage Scope emphasizes stable regression coverage of the highest-risk workflows and rules. Non-deterministic or low-value UI cosmetics are excluded from automation and covered by exploratory testing. NEO Reporting – Automation Test Coverage Test Coverage by Area 1.
