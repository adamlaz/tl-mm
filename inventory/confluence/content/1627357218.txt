# Root Cause Analysis: Account 11607728 SQS Retry Storm
# Space: TT
# Page ID: 1627357218

Executive Summary Between and , account 1607728 experienced 40,681 failed API requests due to a cascading failure caused by missing product configuration in Operator API combined with a misconfigured SQS retry mechanism ( errorVisibilityTimeout=0 ). The issue created an infinite retry loop where the same message was processed thousands of times per second. The issue self-resolved when SQS messages expired after the 4-day retention period. Missing Account Data Account 11607728 has no products configured in Operator API Operator API endpoint /accounts/11607728/products returns error code 10920 : "Account products not found" This is a data integrity issue - account exists but has no associated products The account in question is an Enterprise so products should not be associated (?) 2. Queue target : stag-menucore.fifo Producer : enterprise-menu-service data-sync publisher Action implications: Consider adding an Enterprise guard in EMS data-sync publish flow to skip SQS for enterprise accounts Migrated Endpoints and ESB Routing Impact Menu-core performs migration checks for any request that matches configured migrated endpoints. In staging this list is sourced from /etc/menu-core-api/migratedEndPoints.properties : /v1/accounts/:accountId/catalog /v1/accounts/:accountId/data-sync/:messageId /v1/accounts/:accountId/sessions Why this matters : Any request routed to menu-core for these paths (direct or via ESB) triggers ClientMigrationInterceptor . The interceptor always calls Operator /accounts/:id/products , which returns 10920 for Enterprise accounts. That translates into HTTP 500 and can cause retry storms in downstream consumers (e.g., menu-data-mediator), or high error rates if ESB routes catalog/sessions traffic for Enterprise accounts. Implications for ESB routing : If ESB routes Enterprise catalog or sessions requests through menu-core, it will trigger the migration check and the Operator products call on every request. This can amplify error volume and should be considered when deciding which endpoints are routed to menu-core or which endpoints are marked as migrated.
