Release Overview 1.1 High-level Summary of the Release This release introduces support for enterprise-level custom employee PINs in UA Staff. For more details, please refer to REST-18000 f496ff7d-738c-3ddd-a9cd-8b868d553dc0 System Jira . In addition, the release addresses a legacy issue where employee PINs could be duplicated at the account level during account movement operations , ensuring PIN integrity and preventing unintended duplication during account movement operations. 1.2 Has the Impact to the following groups been addressed? 1.2.1 Operators/Ops : Not Applicable Check List Item (Yes / No) Platform has to be restarted No Customers will have a downtime No Change in user journey No Customer notification has to go out No Comms to Ops Leads Needed No 1.2.2 Commercialization Activities Not Applicable Check List Item (Yes / No) Training Provided to CX Operators No Training Manuals Created // Reference Added to section 1.5 No Need Cake University Articles for Merchants No Need to Inform Marketing No Announcements Via UA is Sent No 1.3 Limitations/Callouts : During testing, we observed a limitation where employee PINs assigned to deleted employees cannot be reused by other employees. However, this behavior is consistent with the existing implementation of auto-generated PINs, where PINs assigned to deleted employees are also excluded from reuse.
