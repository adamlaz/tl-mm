Release Overview 1.1 High-level Summary of the Release This release includes enhancements to the Consumer API and ESB, along with platform-level fixes for DoorDash and production issue resolutions. Consumer Api Enhancements REST-17366 f496ff7d-738c-3ddd-a9cd-8b868d553dc0 System Jira The current queue consumer count update is synchronous and can take over 30 seconds to process. This has been changed to an asynchronous approach, so ESB threads no longer need to wait. ESB Log Enhancements REST-17370 f496ff7d-738c-3ddd-a9cd-8b868d553dc0 System Jira Updating wrapper.log configuration: Adjust log rotation to ensure production-safe behavior. Disabling unnecessary appenders: Remove unused file appenders, redundant console appenders, and in-memory appenders. DoorDash Platform Level Fixes REST-17396 f496ff7d-738c-3ddd-a9cd-8b868d553dc0 System Jira Stop retrying store activation requests that fail with 4XX errors. REST-17358 f496ff7d-738c-3ddd-a9cd-8b868d553dc0 System Jira Introduce a specific error code for order failures caused by invalid promised time to help DoorDash troubleshooting and exclude them from quality metrics.
