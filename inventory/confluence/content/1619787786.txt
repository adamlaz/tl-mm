(NEW) Initiative: REST-10522 f496ff7d-738c-3ddd-a9cd-8b868d553dc0 System Jira Updating the existing POS Preview to display the new UI - REST-10945 f496ff7d-738c-3ddd-a9cd-8b868d553dc0 System Jira Revert or Publish Ability for the user to pick from a list of saved and queued changes and allow to Revert (Undo) or Publish. Currently, on EMS when changes are performed, they all get queued up and the user is required to publish in bulk for the changes to reflect on the product touchpoints (POS, OLO, Kiosk). Also, the user does not have a way to undo or revert a change after saving a change. This can be cumbersome if a particular change was done mistakenly, since the user will have to go back and do another change to override the last action and publish Therefore, by giving the ability to view the list of changes in the queue and select which ones need to be published or reverted will add value to the overall menu management experience. Customer Facing Menu Templates - Dine In As a merchants entire menu is already added to menu admin - Categories, Item Names, Descriptions, Price, Images, we want to provide the ability to take all that existing data and be able to display it on a customer facing menu that merchants can print/download and use in their restaruant/website/or social media. Provide the capability for a user to create a customer facing menu pulling the data off menu admin. Provide the user with customer facing menu templates where they can pull the menu data - Category, Item Name, Description, and Price and create a customer facing menu with ease and efficiency so they could export as a PDF and print it if needed.
