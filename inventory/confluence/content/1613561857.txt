Release Overview 1.1 High-level Summary of the Release The Sales by Discounts report was failing with an error when trying to load data for accounts with a large number of transactions (over 100,000 discount records). Why it happened: The system was using a page-by-page approach to fetch records, but the database has a built-in safety limit that prevents fetching beyond page 10 (when each page has 10,000 records). Large accounts with many transactions were hitting this limit. Solution Changed the way the system fetches large amounts of data. Instead of asking for "page 1, page 2, page 3...", the system now uses a "continue from where I left off" approach — similar to using a bookmark instead of counting pages.
