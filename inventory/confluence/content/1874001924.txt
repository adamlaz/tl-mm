Follow a pattern similar to how Door Dash was implemented within the https://bitbucket.org/syscolabs/olo_aggregator_api/src/master/ . Initial Setup The creation of an EzCater subscriber object will be done manually since it only has to be done once per environment. Specifically, this is the Subscriber Create mutation. The subscriberId and webhookSecret should be stored in Keeper so they can be exposed as config items to the https://bitbucket.org/syscolabs/olo_aggregator_api/src/master/ repo. Data Persistence The following pieces of information need to be persisted for each CAKE merchant EZCater Tab Enabled - This is what gets set by the “Go Live” switch in the Integration drawer EZCater Real Store ID - This is the ezcater-caterer-id corresponding to the CAKE merchant. EZCater (or MadMobile) will manually provide the real store id. EZCater Test Store ID - This is an instance of ezcater-caterer-id specifically for testing menu creation. EZCater manually will provide a test store along with each real store. In some cases multiple, related merchants may have the same test store; this won’t be an issue, it will just have various CAKE merchants who happen to have the same value in this field. EZCater Receive Orders - This is what gets set by the “EZ Cater” switch in the EZ Cater tab EZCater Order Update Cutoff Window - This value is displayed on the UI in hours. EZCater Catering Prep Lead Time - This value is displayed on the UI in minutes.
