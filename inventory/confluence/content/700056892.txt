Release Overview 1.1 High-level Summary of the Release This migration will cover the capability to migrate existing single hierarchy enterprise merchants from EMS 1.0 to EMS 2.0 with no impact on their menu data, reports, staff, 3rd party integrations, and general day to day operations. All existing variations will be maintained. Special Note: Data disrecpencies due to EMS 1.0 bugs have been corrected for EMS 2.0. Further details available in the callout section. After the migration, the merchants will have access to the new features of the EMS 2.0 such as - 86'ing and activity logs. 1.2 Has the Impact to the following groups been addressed? 1.2.1 Operators/Ops : Check List Item (Yes / No) Platform has to be restarted 127 incomplete No Customers will have a downtime 129 incomplete No Change in user journey 131 incomplete No Customer notification has to go out 133 incomplete No Comms to Ops Leads Needed 135 incomplete Yes 1.2.2 Commercialization Activities Check List Item (Yes / No) Training Provided to CX Operators 137 incomplete Training Manuals Created // Reference Added to section 1.5 138 incomplete Need Cake University Articles for Merchants 139 incomplete No Need to Inform Marketing 140 incomplete Announcements Via UA is Sent 141 incomplete No 1.3 Limitations/Callouts : Callout: These callouts will not impact the merchants day to day operations (These callouts are bugs that are on EMS 1.0 and the data has been correct for EMS 2.0) POS/OLO/Kiosk Check Boxes will be marked accurately as per the catalog Any item that is without a category assigned will be in the uncategorized category Modifier options in the ungrouped section will not be migrated as these are discarded modifier options. (similar approach was taken for single lcoation migrations). However, if there are any modifier groups within the ungrouped list, they will be migrated along with the modiifer options assigned to them Modifier groups that had the min/max value set to "1/0" will have the max value updated to the maximum number of options in the modifier group.
