Release Overview 1.1 High-level Summary of the Release This release updates the CAKE–Paytronix integration to leverage Paytronix’s native out-of-stock functionality instead of removing and re-adding menu items when they are marked unavailable. When a menu item is marked Out-of-Stock in CAKE, the system will now update the item in Paytronix using the is_sold_out: true flag through the API. When the item is marked back In-Stock, the flag will be reverted accordingly. This approach prevents the removal and re-creation of menu items in Paytronix, ensuring that item images remain intact. 1.2 Has the Impact to the following groups been addressed? 1.2.1 Operators/Ops : Not Applicable Check List Item (Yes / No) Platform has to be restarted No Customers will have a downtime No Change in user journey No Customer notification has to go out No Comms to Ops Leads Needed No 1.2.2 Commercialization Activities Not Applicable Check List Item (Yes / No) Training Provided to CX Operators No Training Manuals Created // Reference Added to section 1.5 No Need Cake University Articles for Merchants No Need to Inform Marketing No Announcements Via UA is Sent No 1.3 Limitations/Callouts : Not Applicable 1.4 Known Issues : Not Applicable 1.5 References REST-17498 f496ff7d-738c-3ddd-a9cd-8b868d553dc0 System Jira 2.
